Forbidden inheritance.

Forbidden to in herit from specific exemplars.

Software can be structured in such a way that product A provides the base software for the datamodel, product B provides all the GUI related software, and product C provides the headless (job server) tasks. In this case, you want all your record exemplars in product A, as these are used by product B and C.

Please do note that it might be required to add both the sw and user packages, as the actual reference will be resolved at compile time. The check cannot determine the current state of the session. To be exact, it is recommended that you specify the parent through the global reference, i.e., @sw:rwo_record

Non-compliant Code Example

A non-compliant code exemplar can be as follows, when it is forbidden to inherit from sw:rwo_record.

def_slotted_exemplar(
    :test_exemplar,
    {},
    :rwo_record)
